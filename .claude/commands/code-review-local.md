## チェックリスト

### 機能要件
- [ ] 仕様・要件を満たしている
- [ ] エッジケースを考慮している
- [ ] 既存機能へのデグレがない

### 非機能要件
- [ ] パフォーマンスへの影響を確認した
- [ ] ログ出力が適切である
- [ ] テストを書いた / 既存テストがパスする

### セキュリティ / コンフィグ
- [ ] 機密情報がコードに含まれていない
- [ ] 入力バリデーションが適切である
- [ ] 設定値が環境変数・設定ファイルで管理されている

### コード品質
- [ ] 命名が分かりやすい
- [ ] 重複コードがない（DRY原則）
- [ ] コメント・ドキュメントが適切に書かれている
- [ ] TODO/FIXME が残っていない（or チケット化済み）
