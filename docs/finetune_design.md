# ファインチューニング基盤 概要設計

目的
- OpenAI に依存しないファインチューニング基盤を構築し、RAG で利用可能にする。

初期スコープ
1. 学習データの自動収集・サニタイズ（履歴・面接・スコア）
2. JSONL エクスポートユーティリティ（PII マスキング）
3. LoRA/QLoRA を用いた学習スケルトン（ドキュメント付き）
4. 設計ドキュメントと実行手順の準備

方針と理由
- まずはデータ収集とフォーマット整備に注力し、モデル選定やGPU運用は段階的に実施する。
- LoRA を初期手法に採用することで小規模な計算資源でも適用しやすい。

次の作業
- データ収集 API の実装（rag サービス内にエンドポイント追加）
- 学習用パイプライン（Trainer スクリプト）の作成
- テストと CI の整備（データ品質チェック）

リスクと課題
- 個人情報（PII）管理: 法的要件に従い匿名化を徹底すること
- コスト: GPU/ストレージの試算が必要
