# Issue: 要件定義: 選考ステータスの状態遷移設計

## 概要

初めて上流工程を担当するタスクとして、ユーザーの応募後に発生する選考ステータスの状態遷移を定義する。

採用支援SaaSとして、応募・書類選考・面接・内定・辞退・不採用などの業務フローを整理し、後続の画面設計・API設計・実装に進めるための仕様書を作成する。

## 背景

選考結果は、マッチング精度改善、企業プロファイル更新、スコアキャリブレーション、集合知レコメンドに利用される重要データである。
そのため、ステータスを自由入力や曖昧な更新にせず、業務上許可される状態遷移として整理する必要がある。

## 成果物

- 選考ステータス一覧
- 状態遷移図
- 遷移可能・遷移不可ルール
- ユーザー・管理者・システムの操作権限
- 画面要件
- API要件
- エラー条件
- 未決事項・確認事項

仕様書:

- `docs/requirements/application-status-transition.md`

## 対象範囲

### 対象

- ユーザーの応募後の選考管理
- ステータス更新ルール
- 辞退・内定承諾・不採用などの終了状態
- 画面とAPIに必要な主要項目

### 対象外

- 企業推薦ロジック
- AI面接の質問生成
- 職務経歴書レビューの評価ロジック
- スコアキャリブレーション計算式
- メール通知文面の詳細

## 完了条件

- [ ] 選考ステータスの種類と意味が整理されている
- [ ] 状態遷移図が作成されている
- [ ] 禁止する遷移が明記されている
- [ ] ユーザー・管理者・システムごとの操作権限が整理されている
- [ ] 画面で必要な表示項目・操作が整理されている
- [ ] APIで必要な主要エンドポイントが整理されている
- [ ] レビュー観点として未決事項が洗い出されている
